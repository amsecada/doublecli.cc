# DoubleClicc Agent Guide

## Mission
This repo powers the DoubleClicc website. The goal is to position DoubleClicc as a premium systems + AI automation partner for service businesses.

## Where to read for context
Before making ANY content or marketing changes, read through the readme/ folder; it contains critical information / context on the website.

## Content principles
- Narrative > bullet points
- Focus on systems, not tools
- Emphasize business outcomes over features
- Avoid generic AI language

## Case study rules
- Use only approved facts from case study docs
- Preserve tone: story-driven, high-authority
- Do not simplify into generic marketing copy

## When working on website pages
- Homepage = concise, punchy
- Case studies = longform, narrative
- Reuse existing phrasing where possible
